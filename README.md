# 声録 (koe-roc) - M5StickC 音声録音デバイス

声録（こえろく）は、M5StickC を用いて音声を継続的に録音し、無音空間を検知して自動的にファイルを分割・保存する実験用デバイスです。
将来的にはドック接続による自動データ吸い出しや、LLMを用いたデータ分析への統合を目指しています。

## 1. 開発環境のセットアップ (Nix Flake)

本プロジェクトは **Nix Flake** を使用して開発環境（PlatformIO CLIなど）を完全に再現できるように構成されています。

### 開発シェルへの入り方
以下のコマンドを実行することで、PlatformIOがインストールされたシェルに入ることができます。
```bash
nix develop
```

## 2. ビルド & 実機への書き込み手順

### ビルドの実行
コードをコンパイルするには、開発シェル内で以下を実行します。
```bash
pio run
```

### ファームウェアのアップロード（実機書き込み）
M5StickCをUSBケーブルでPCに接続し、以下を実行します。
```bash
pio run --target upload
```

### シリアルモニターの開始（ログ確認とデバッグ）
実機のログを確認し、マイクの感度や音量レベルを確認するには以下を実行します。
```bash
pio device monitor
```
※ モニターを終了するには `Ctrl + ]` を押します。

## 3. デバイスの操作方法

本デバイスは、前面ボタンの故障時でも快適に操作できるよう、**側面ボタン（Button B）のみ**ですべての操作を完結できるように設計されています（電源ボタン `BtnPWR` および前面ボタン `BtnA` は予備の短押し用として機能します）。

ボタンの押し込み時間（画面にリアルタイムでゲージが表示されます）によって動作が分岐します：

*   **短押し（1秒未満で離す）**:
    *   **待機中 (IDLE)**: 録音を開始します。
    *   **録音中 (RECORDING)**: 録音を一時停止します。
    *   **一時停止中 (PAUSED)**: 録音を再開します。
    *   **APモード中 (WIFI_SERVER)**: APモードを終了して待機中 (IDLE) に戻ります。
*   **長押し（1秒以上〜5秒未満で離す）**:
    *   **自動アップロードモード**:
        *   録音中の場合は録音を停止・保存したあと、保存されたすべてのWAVファイルをPCのアップロード用サーバーへ自動転送します。
        *   転送が成功したファイルは自動的に本体（LittleFS）から削除されます。
*   **超長押し（5秒以上押し続ける）**:
    *   **設定・個別ダウンロード用APモード (WIFI_SERVER)**:
        *   WiFiアクセスポイントを起動します。画面表示が `WIFI` に切り替わり、PCやスマホからブラウザ経由で設定変更やファイルのダウンロードが可能です。もう一度5秒以上長押しするか、短押しすると終了します。

## 4. 自動アップロード機能と設定方法

WAVファイルを自動的にPCへアップロードするための手順です。

### 1. PC側アップロードサーバーの起動
PC（またはサーバー）で付属の Flask サーバーを起動します。
```bash
# 必要ライブラリのインストールと起動（nix develop 環境下であれば uv が利用可能）
uv run python server.py
# もしくは
python3 -m pip install flask
python3 server.py
```
* 起動すると、ポート `8080` で待機状態になります。

### 2. デバイス側のWiFi設定
1. デバイス（M5StickC）を **5秒以上超長押し** してAPモードを起動します。
2. PCやスマホから SSID `koe-roc` (PASS: `12345678`) に接続し、ブラウザで `http://192.168.4.1/` にアクセスします。
3. 設定画面が表示されるので、接続先の **WiFi SSID**、**パスワード**、および PCの **アップロード先URL** (例: `http://<PCのIPアドレス>:8080/upload`) を入力して保存します。
4. 設定が保存されると、次回以降の「自動アップロード」でこの設定が使われます。

### 3. アップロードの実行
* デバイスを **1〜5秒間長押し** すると、設定したWiFiに自動で接続され、未送信の全WAVファイルが順次PCへアップロードされ、成功後にローカルから安全に削除されます。

## 5. 画面表示 (UI) の見方

*   **上部バー**: 動作状態を表示。
    *   `REC`（赤背景）: 録音中
    *   `PAUSE`（オレンジ背景）: 一時停止中
    *   `IDLE`（グレー背景）: 停止・待機中
    *   右端にはバッテリー残量（%）が表示されます。
*   **中央エリア**:
    *   現在録音中のファイル名（RTCの時刻から自動生成: `rec_HHMMSS.wav`）
    *   現在のファイルにおける録音時間（分:秒）
*   **下部エリア**:
    *   **音量レベルメーター**: マイクから入力された音量（絶対値平均値）をリアルタイムにバーで表示。
        *   無音閾値（`SILENCE_THRESHOLD` = 300）を超えている間は **緑色**、下回っている間は **青色** で表示されます。
    *   **ストレージ情報**: 内蔵Flash（LittleFS）の残り空き容量（MBとパーセンテージ）を表示。

## 6. 無音検知と自動分割ロジック

長時間の録音によるファイル肥大化を防ぐため、以下の条件が揃った時に自動的にファイルを分割（現在のファイルを保存して新規ファイルを作成）します。

*   **自動分割条件**:
    1.  現在のファイルで **30秒以上** 録音している。
    2.  音量レベルが `SILENCE_THRESHOLD` (300) を下回る状態（青色のメーター）が **1.5秒** 以上継続する。

### 閾値の調整について
周囲の環境ノイズが大きい場合やマイク感度を調整したい場合は、`src/main.cpp` の以下の定数を調整してください：
*   `SILENCE_THRESHOLD`: 無音と判定する音量の上限。
*   `SILENCE_DURATION_MS`: 無音と判定してから実際にファイルを分割するまでの時間（ミリ秒）。
*   `MIN_RECORD_DURATION_MS`: ファイル分割を行う最低の録音時間（ミリ秒）。
