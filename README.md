# 声録 (koe-roc) - M5StickC 音声録音デバイス

声録（こえろく）は、M5StickC を用いて音声を継続的に録音し、無音空間を検知して自動的にファイルを分割・保存する実験用デバイスです。
将来的にはドック接続による自動データ吸い出しや、LLMを用いたデータ分析への統合を目指しています。

## 1. 開発環境のセットアップ (Nix Flake)

本プロジェクトは **Nix Flake** を使用して開発環境（PlatformIO CLIなど）を完全に再現できるように構成されています。

### 開発シェルへの入り方
以下のコマンドを実行することで、PlatformIOがインストールされたシェルに入ることができます。
```bash
nix develop
```

## 2. ビルド & 実機への書き込み手順

### ビルドの実行
コードをコンパイルするには、開発シェル内で以下を実行します。
```bash
pio run
```

### ファームウェアのアップロード（実機書き込み）
M5StickCをUSBケーブルでPCに接続し、以下を実行します。
```bash
pio run --target upload
```

### シリアルモニターの開始（ログ確認とデバッグ）
実機のログを確認し、マイクの感度や音量レベルを確認するには以下を実行します。
```bash
pio device monitor
```
※ モニターを終了するには `Ctrl + ]` を押します。

---

## 3. デバイスの操作方法

本デバイスは、前面ボタンの故障時でも快適に操作できるよう、**側面ボタン（Button B）のみ** ですべての操作を完結できるように設計されています。

*   **シングルクリック（短押し 1回）**:
    *   録音の開始 / 一時停止 (Pause / Resume) をトグルします。
    *   **WiFiサーバーモード中**に押すと、WiFiサーバーを停止して `IDLE`（待機）状態に戻ります。
*   **長押し（1秒以上押す）**:
    *   **録音中 / 一時停止中**：録音を完全に停止し、現在のファイルを保存して `IDLE`（待機）状態に戻ります。
    *   **待機（IDLE）中**：**WiFiサーバーモード** を開始します。

### WiFi経由でのデータ取り出し手順
1.  デバイスが `IDLE`（待機中）のときに、側面ボタンを **1秒以上長押し** します。
2.  画面表示が紫色の `WIFI` に切り替わり、以下の接続情報が表示されます。
    *   **SSID**: `koe-roc`
    *   **PASS**: `12345678`
    *   **URL**: `http://192.168.4.1/`
3.  PCまたはスマホのWiFi接続設定から、SSID `koe-roc` に接続します（パスワード: `12345678`）。
4.  Webブラウザを開き、アドレスバーに `http://192.168.4.1/` と入力してアクセスします。
5.  ブラウザ上に録音ファイルの一覧が表示され、**「Download」**をクリックしてファイルをPCへ取得できます。また、不要なファイルは「Delete」で消去可能です。
6.  取り出しが終わったら、M5StickCの側面ボタンを **1回** 押すと、WiFi接続が終了し元の `IDLE` 状態に戻ります。



## 4. 画面表示 (UI) の見方

*   **上部バー**: 動作状態を表示。
    *   `REC`（赤背景）: 録音中
    *   `PAUSE`（オレンジ背景）: 一時停止中
    *   `IDLE`（グレー背景）: 停止・待機中
    *   右端にはバッテリー残量（%）が表示されます。
*   **中央エリア**:
    *   現在録音中のファイル名（RTCの時刻から自動生成: `rec_HHMMSS.wav`）
    *   現在のファイルにおける録音時間（分:秒）
*   **下部エリア**:
    *   **音量レベルメーター**: マイクから入力された音量（絶対値平均値）をリアルタイムにバーで表示。
        *   無音閾値（`SILENCE_THRESHOLD` = 300）を超えている間は **緑色**、下回っている間は **青色** で表示されます。
    *   **ストレージ情報**: 内蔵Flash（LittleFS）の残り空き容量（MBとパーセンテージ）を表示。

## 5. 無音検知と自動分割ロジック

長時間の録音によるファイル肥大化を防ぐため、以下の条件が揃った時に自動的にファイルを分割（現在のファイルを保存して新規ファイルを作成）します。

*   **自動分割条件**:
    1.  現在のファイルで **30秒以上** 録音している。
    2.  音量レベルが `SILENCE_THRESHOLD` (300) を下回る状態（青色のメーター）が **1.5秒** 以上継続する。

### 閾値の調整について
周囲の環境ノイズが大きい場合やマイク感度を調整したい場合は、`src/main.cpp` の以下の定数を調整してください：
*   `SILENCE_THRESHOLD`: 無音と判定する音量の上限。
*   `SILENCE_DURATION_MS`: 無音と判定してから実際にファイルを分割するまでの時間（ミリ秒）。
*   `MIN_RECORD_DURATION_MS`: ファイル分割を行う最低の録音時間（ミリ秒）。
